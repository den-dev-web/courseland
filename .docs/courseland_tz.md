# Техническое задание (ТЗ)
# **CourseLand — платформа онлайн‑курсов (лендинг + каталог)**

---

## **1. Общая информация о проекте**
**Название:** CourseLand — веб‑платформа для выбора и покупки онлайн‑курсов.  
**Тип:** лендинг + каталог курсов + карточка курса.  
**Цель:** показать навык работы с лендинг‑структурой, компонентными карточками, слайдерами, формой подписки, FAQ и фильтрами.

**Стек:**
- HTML5 + семантическая структура
- CSS (ITCSS + BEM)
- Vanilla JavaScript
- Оптимизированные изображения (WebP/AVIF)

Проект должен выглядеть как коммерческий лендинг крупной EdTech‑платформы.

---

## **2. Структура страниц**
`index.html` - главная страница.

`course.html` — карточка курса.

### **2.1. Главная страница (index.html)**
Секции:
1. Header
2. Hero
3. Категории курсов
4. Каталог курсов
5. Преимущества платформы
6. Слайдер «Лучшие курсы недели»
7. Блок «Как это работает»
8. Отзывы
9. FAQ
10. Форма подписки
11. Footer

### **2.2. Страница курса (course.html)
Секции:
- Hero курса (заголовок, рейтинг, длительность)
- Описание
- Программа курса (аккордеон)
- Информация об авторе
- Отзывы
- CTA‑кнопка «Записаться»

---

## **3. Основной функционал**
### **3.1. Каталог курсов**
Каждый курс должен содержать:
- Превью‑картинку (через `<picture>`)
- Название
- Категорию
- Краткое описание
- Продолжительность
- Цена / Бесплатно
- Рейтинг (иконки звёзд)

Переход по клику → `course.html`.

### **3.2. Фильтрация каталога (JS)**
Фильтры:
- По категории
- По цене (бесплатные / платные)
- По длительности (короткие / длинные)
- По рейтингу

Фильтрация — без перезагрузки страницы.

### **3.3. Категории курсов**
- Горизонтальный список категорий
- Активная категория подсвечивается
- При клике — прокрутка к каталогу + применённый фильтр

### **3.4. Слайдер «Лучшие курсы недели»**
- Можно использовать Swiper или собственную маленькую реализацию
- На мобильных — 1 карточка, на таблетах — 2, на десктопе — 3

### **3.5. FAQ (аккордеон)**
- Стандартный аккордеон
- aria‑атрибуты (`aria-expanded`)

### **3.6. Форма подписки**
Поля:
- Email

Функционал:
- Валидация email
- Состояние при отправке
- Отправка через `fetch` → mock endpoint
- Сообщения success / error (aria‑live)

---

## **4. Адаптивность**
Подход: **mobile‑first**.

### Breakpoints:
- <600px — мобильные
- 600–1024px — планшеты
- >1024px — десктоп

### Основные адаптивные изменения:
- Каталог — 1 → 2 → 3–4 колонки
- Слайдер — 1 → 2 → 3 карточки
- Меню — бургер на мобилке
- Hero — вертикальный стек на мобиле, двухколоночный на десктопе

---

## **5. UX и анимации**
- Плавное появление секций при скролле (scroll reveal)
- Hover‑состояния карточек и кнопок
- Анимация слайдера
- Skeleton loading каталога (по желанию)
- Переходы 150–250ms
- prefers‑reduced‑motion — отключение анимаций

---

## **6. Доступность (a11y)**
- Семантические структуры секций
- Правильные alt‑тексты
- Управляемый с клавиатуры бургер‑меню
- Доступный аккордеон FAQ
- Правильные лейблы у форм
- aria‑live для уведомлений

---

## **7. Оптимизация**
- `<picture>` + WebP/AVIF
- Lazy‑loading изображений
- aspect‑ratio для карточек
- Минифицированные CSS и JS
- Цель по Lighthouse: Accessibility — 100 / Performance — 90+
